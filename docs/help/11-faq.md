# 常见问题

## 订单相关

**Q：为什么订单不能取消？**

取消订单有两个前置条件：未发货（发货状态为「待发货」）且无收款。已发货的订单请走销售退货流程；已收款的订单请先完成退款，再取消。

**Q：订单什么时候自动变成「已完成」？**

全部发货（发货状态 = 全部发货）且全部收款（收款状态 = 已收款）后，订单自动流转为「已完成」，无需手工操作。

**Q：待发货/待收款卡片为什么没有某个订单？**

已取消的订单不计入待办；已完成的订单（全部发货且全部收款）自然排除。这是设计口径，避免已终结的订单干扰待办。

## 发货与库存相关

**Q：发货时提示库存不足怎么办？**

系统按 FIFO 分配批次，库存不足时无法超发。请先安排采购入库，或与客户协商调整发货数量。

**Q：发货弹窗里的预估成本是怎么算的？**

按 FIFO 批次分配结果计算：各批次分配数量 × 批次单位成本之和，并随本次发货数量实时联动。提交后后端按相同规则重新计算落库。

**Q：库存预警的阈值在哪里设置？**

优先在「商品 → 型号」上设置最低库存；未设置的型号使用系统配置中的全局阈值。停用商品/型号不参与预警。

## 收付款相关

**Q：支持哪些币种？**

支持美元（USD）与人民币（CNY）。订单、收款、退款均可选择币种，系统按汇率折算人民币金额用于统计。

**Q：客户多付/少付了怎么办？**

按实际到账金额登记收款即可，订单会保持「部分收款」状态直至收满。多付部分可与财务确认后登记退款或留作预收。

## 系统使用相关

**Q：页面打开后之前的查询条件还在吗？**

系统会缓存已打开的页面（标签页机制），切换回来时保留之前的状态。关闭标签页后重新打开则恢复默认。

**Q：为什么有的按钮我看不到？**

按钮与菜单按角色权限显示。如确需使用，请联系管理员在「角色管理」中分配相应权限。

**Q：在哪里看自己的操作记录？**

管理员可在「系统管理 → 操作日志」中查看全部操作记录，支持按人员与时间筛选。
