# 常见问题

## 订单相关

**Q：为什么订单不能编辑？**

仅「待发货」状态的订单可以编辑。一旦发过货（部分发货或全部发货），订单即锁定不可修改。如需调整，请通过退货流程处理后重新操作。

**Q：为什么订单不能取消？**

取消有两个前置条件：未发货（发货状态为「待发货」）且无收款。已发货的订单请走客户退货流程；已收款的订单请先完成退款，再取消。

**Q：「终止」和「取消」有什么区别？**

- **取消**：针对待发货订单，整单作废
- **终止**：针对部分发货的订单，客户弃购未发部分——系统将订单数量收缩为已发数量，总金额同步减少，超收金额自动退款

**Q：订单什么时候自动变成「已完成」？**

全部发货（发货状态 = 全部发货）且全部收款（收款状态 = 已收款）后自动流转，无需手工操作。已完成的订单如果发生退货换货/补发不退货导致发货状态下降，会重新打开为「进行中」。

**Q：为什么编辑订单时提示「已收款超过新总金额」？**

修改商品明细后货物总金额减少，但已收金额超过了新的总额。请先通过「退款」退还超出部分，再修改订单。

**Q：订单的「销售利润」为什么是空的或显示「—」？**

利润 = 订单金额 − 博主佣金 − 已退款 − 直接退款 − 产品成本 − 额外成本。其中产品成本仅统计已发货部分（FIFO（先进先出）成本），未发货的订单产品成本为 0 但利润仍可计算。如果显示「—」表示数据尚未产生（通常是无明细的异常情况）。

## 发货与库存相关

**Q：发货时提示「库存不足：需要 X，可用 Y」怎么办？**

系统按 FIFO（先进先出）分配批次，不允许超发。解决方案：先安排采购入库补充库存，或减少本次发货数量。

**Q：发货弹窗里的预估成本是怎么算的？**

按 FIFO 批次分配结果计算：从最早入库的批次开始依次分摊本次发货数量，各批次分配数量 × 批次单位成本之和。预估成本随发货数量输入实时联动。提交后系统按相同规则重新计算并保存最终结果。

**Q：什么是「补发」类型？怎么操作？**

补发用于「补发不退货」场景（如货物损坏直接发新货）。完整流程：

1. 先在客户退货中创建「补发不退货」类型的退货单
2. 再在发货管理中选择「补发」类型创建发货单
3. 补发明细的销售金额按 0 计（纯成本单据）

**Q：库存预警的阈值在哪里设置？**

两级优先级：

1. 型号级：商品管理 → 型号 → 最低库存预警值（留空=用全局，0=零库存才预警，负数=不预警）
2. 全局级：系统参数中的「库存预警阈值」（默认 0，即库存为 0 才预警）

停用商品/停用型号/已删除型号不参与预警。

**Q：库存调整需要审批吗？**

不需要。库存调整提交即生效，无审批流程。请确认数量无误后再提交，所有调整都会记录在库存流水中可追溯。

**Q：库存调整时「成本来源」怎么选？**

仅在「增加库存且未指定批次」时需要选择：

- 近一年加权平均成本：适合常规盘盈
- 剩余库存加权平均成本：适合当前有库存参考的情况
- 最新采购记录成本：适合刚采购过的商品
- 手动输入：以上都不适用时自行填写

减少库存时不需要选成本来源（按 FIFO 批次实际成本出库）。

## 收付款相关

**Q：收款时币种可以选吗？**

不可以。收款币种固定为订单币种，系统按收款日期自动查询汇率换算双币种入账。

**Q：退款后利润为什么没有减少那么多？**

直接退款使用订单原始汇率换算（而非退款日汇率），保证「全额退款 → 利润归零」不产生虚增汇兑损益。

**Q：客户多付了怎么办？**

收款金额不能超过订单总金额，系统会拒绝超额收款。多付部分请与财务确认后另行处理（线下退还或留作预收）。

**Q：收款后之前的退款会怎样？**

后续收款会优先冲抵此前的直接退款金额，冲抵后利润相应恢复，提成也会差额重算。

**Q：收/退款单可以修改或删除吗？**

不可以。收/退款单提交后即为终态，如需冲正请登记一笔反向单据。

## 退货相关

**Q：四种退货类型怎么选？**

| 场景 | 选择 |
| --- | --- |
| 客户退货 + 退钱 | 退货退款 |
| 客户退货 + 后续补发新货 | 退货换货 |
| 货不退 + 只退钱 | 仅退款 |
| 货不退 + 直接补发新货 | 补发不退货 |

恢复库存和退回货款由类型自动决定，无需手工设置。

**Q：退货退款后收款状态为什么没变？**

客户退货触发的退款不扣减订单「已收金额」，而是记入独立的「退货退款/仅退款」字段。收款状态保持不变，退款金额参与利润计算。

**Q：退货换货后订单为什么从「已完成」变回「进行中」？**

退货换货降低了客户持有量，使发货状态从「全部发货」回退为「部分发货」，订单重新打开等待补发。补发完成后会再次自动变为已完成。

## 采购相关

**Q：采购退货时为什么可退数量比已入库数量少？**

可退数量取「已入库 − 已退货」与「当前库存可用量」两者中的较小值。如果部分库存已被销售发货消耗，实际可退数量以当前库存可用量为准。

**Q：采购单关闭后还能恢复吗？**

不能。关闭为终态操作，请确认后再执行。如需重新采购，请新建采购单。

## 汇率相关

**Q：创建单据时提示「未找到汇率数据」怎么办？**

三种解决方式：

1. 在汇率管理中手动新增对应日期的 USD→CNY 汇率
2. 点击汇率管理页的「同步汇率」按钮自动拉取最近 30 天数据
3. 调整单据日期到已有汇率记录的日期

**Q：汇率可以手工修改吗？**

可以。在汇率管理页编辑对应记录的汇率值即可。修改后新创建的单据会使用新值，已创建的历史单据不受影响（汇率已固化在单据中）。

## 提成相关

**Q：订单完成了为什么没有产生提成？**

可能原因：

- 订单未关联销售员
- 销售员已停用（停用时不计提）
- 销售利润 ≤ 0（亏损订单不产生提成）
- 提成比例设置为 0

**Q：退款后提成会怎样？**

- 退款使订单从「已完成」退回「进行中」→ 提成全额撤销
- 退款后订单仍为「已完成」→ 提成按新利润差额重算（可能减少，也可能因冲抵后利润上升而补提）

**Q：月度结算为什么不能选当月？**

当月尚未结束，提成可能还会变动（退款冲回等），只能结算已结束的历史月份。且必须按顺序结算，不能跳月。

## 系统使用相关

**Q：页面打开后之前的查询条件还在吗？**

系统最多保留 10 个已打开页面的状态（标签页机制），切换回来时保留之前的搜索条件和滚动位置。关闭标签页后重新打开则恢复默认。如果数据可能过时，右键标签选择「刷新当前」。

**Q：为什么有的按钮我看不到？**

按钮按角色权限控制显示。如确需使用，请联系管理员在「角色管理 → 分配权限」中勾选对应的按钮权限。

**Q：为什么驾驶舱/订单列表看不到全局数据？**

需要管理员为您分配「驾驶舱全局查看」或「全部订单查看」权限。无权限时固定为个人视图，且需要账号绑定了销售员档案才能看到数据。

**Q：忘记密码怎么办？**

联系管理员在「用户管理」中点击「重置密码」，设置新密码后通知您。

**Q：修改密码后为什么要重新登录？**

安全设计：修改密码成功后系统强制登出，需使用新密码重新登录，确保之前的登录状态立即失效。

**Q：如何开启库存预警邮件通知？**

在「系统管理 → 系统参数」中，将「库存预警邮件通知」设为 true，并在「库存预警收件邮箱」中填写收件邮箱（多个用英文逗号分隔）。两个条件都满足后，库存首次从高于阈值降至阈值以下时系统会自动发送一封汇总邮件，列出所有触发预警的商品、型号、当前库存和阈值。库存恢复后再次降至阈值以下会再次通知。邮件发送失败不影响正常业务操作。

**Q：操作日志里为什么查不到某条记录？**

操作日志只记录写操作（新增/修改/删除），查询类操作不记录。另外模块和操作类型筛选为精确匹配，建议直接按时间范围或操作人查询。
