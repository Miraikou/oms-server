# 采购管理

采购管理覆盖向供应商补货的全过程：采购下单 → 到货入库 → 采购退货。涉及三个页面：采购订单管理、入库记录、采购退货管理。

> 【图片占位：采购订单列表页】

## 采购订单

### 状态说明

| 状态 | 含义 |
| --- | --- |
| 待入库 | 已下单，尚未到货 |
| 部分入库 | 部分明细已入库 |
| 全部入库 | 全部明细入库完成 |
| 已关闭 | 手动关闭，不再跟踪 |

另有独立的「退货状态」维度：未退货 / 部分退货 / 全部退货。

### 搜索条件

- 采购单号（模糊匹配）、供应商下拉
- 状态（**多选**）、退货状态（单选）
- 采购日期范围

> 驾驶舱「待入库采购」卡片点击后会跳转并自动筛选「待入库 + 部分入库」。

### 操作按钮

| 按钮 | 出现条件 |
| --- | --- |
| 详情 | 始终可见 |
| 编辑 | 仅待入库 |
| 入库 | 未全部入库且未关闭 |
| 退货 | 部分/全部入库 且 未全部退货 |
| 关闭 | 未全部入库且未关闭 |

### 新建采购单

> 【图片占位：新建采购单弹窗】

1. 选择供应商（仅显示启用的供应商）、采购日期（默认当天）
2. 选择币种（默认 CNY），汇率按采购日期**自动查询**（只读）
3. 添加采购明细：商品（必选）、型号（可选）、数量（正整数）、单价；金额为只读自动计算
4. 提交后采购单进入「待入库」

> 同一采购单内不允许出现重复的「商品+型号」组合。

### 编辑采购单

**仅「待入库」状态可以编辑。** 已入库的采购单不可修改。编辑时明细整体替换，系统重新计算总金额。

### 关闭采购单

长期未到货或作废的采购单可执行「关闭」：

- 已全部入库的无需关闭；已关闭的不可重复操作
- 关闭后不再参与待入库统计，也不可再入库

### 采购单详情

点击「详情」查看：基本信息（供应商、日期、币种、汇率、总金额、状态）、采购明细（含每行的已入库、已退货、**可入库**数量）、入库记录列表。

## 采购入库

入库单号以 `RK` 开头。**入库单不能独立创建**，只能从采购订单的「入库」操作产生。

> 【图片占位：入库操作弹窗（含「全部入库」按钮）】

### 入库操作

1. 在采购订单列表对目标单点击「入库」
2. 弹窗仅列出可入库行（采购数量 > 已入库），填写本次入库数量（不超过可入库数量）
3. 可点击「全部入库」一键填满所有余量
4. 填写入库日期（必填）、备注后提交

### 提交后的系统处理

1. 生成入库单与入库明细
2. **每条入库明细生成一个库存批次**（批次号 `BT`），批次成本锁定为本次入库单价，一经创建不可修改
3. 库存可用量与实际库存同步增加，生成库存流水（业务类型：采购入库）
4. 采购单状态自动更新：全部明细入库完成 → 全部入库，否则 → 部分入库

> 批次成本是后续销售发货 FIFO 成本核算的基础，请确保入库单价填写正确。

### 入库记录页

「入库记录」页面可查询所有入库单：

- 搜索：入库单号（精确匹配）、采购单号（精确匹配）、入库日期范围
- 点击详情查看入库明细（商品、入库数量、单价、金额）

## 采购退货

供应商来货有质量问题时，通过采购退货处理。退货单号以 `PT` 开头。

> 【图片占位：新建采购退货弹窗】

### 新建采购退货

- 方式一：在采购订单列表对已入库的采购单点击「退货」（自动带入）
- 方式二：采购退货管理页点击「新建采购退货」，手动选择采购单（仅列出部分/全部入库的单）

操作步骤：

1. 选择退货日期（默认当天）、填写退货原因与备注
2. 在可退货商品表中填写退货数量
3. 提交

### 可退数量

可退数量取「已入库 − 已退货」与「当前库存可用量」两者中的较小值。

> 即使采购单上还有可退余量，如果对应库存已经被销售发货消耗，实际可退数量会以当前库存可用量为准。

### 提交后的系统处理

1. 生成采购退货单
2. **按 FIFO（先进先出）从最早入库的批次开始扣减可用库存**（不追溯原采购批次），批次耗尽自动标记，生成库存流水（业务类型：采购退货出库）
3. 采购明细已退货数量累加，退货状态自动更新

### 采购退货列表

搜索：退货单号、采购单号、退货时间范围。点击详情查看退货明细。

## 与驾驶舱/库存的联动

- 驾驶舱「待入库采购」卡片统计待入库 + 部分入库的采购单数量
- 入库增加库存批次 → 销售发货按 FIFO 消耗批次 → 采购退货按 FIFO 扣减批次
- 库存不足时销售发货会报错，请及时关注库存预警并安排采购补货
