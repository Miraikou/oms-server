# 基础资料

基础资料是业务单据的支撑数据，建议在正式开单前完成维护。所有基础资料页面的操作方式一致：搜索 → 列表 → 新建/编辑/停用启用。

> 通用规则：停用后的资料不会出现在业务单据的下拉选项中（新单据选不到），但历史单据的引用不受影响、照常显示。

## 供应商

维护供应商基本信息，供应商编号由系统自动生成（`GYS` + 流水号）。

**字段**：供应商名称（必填，唯一）、联系人、联系电话、地址、备注。

**唯一性**：供应商名称不可重复，重复时提示「数据已存在，请勿重复添加」。

**停用影响**：停用后新建采购单时无法选择该供应商。

## 商品分类

用于商品归类的平铺分类列表。

**字段**：分类名称（必填）、排序号（越小越靠前）、备注。

> 分类名称允许重名，无唯一约束。停用后商品表单中无法选择该分类，已关联的商品不受影响。

## 商品

> 【图片占位：商品新建弹窗（含型号内嵌表格与图片上传）】

### 基本信息

**字段**：商品名称（必填）、供应商（必选，仅启用项）、商品分类（可选）、商品图片（拖拽上传，1 张）、备注。

**唯一性**：同一供应商 + 同一分类 + 同一商品名称不可重复。

### 商品型号

每个商品可挂多个型号（规格）。在商品弹窗中直接添加，或在列表中点击「型号」进入型号管理抽屉单独维护。

型号字段：

- 型号名称（必填，同一商品内唯一）
- **最低库存预警值**：可用库存 ≤ 该值时触发预警；留空 = 使用全局阈值（系统参数），填 0 = 库存为 0 时预警，填负数 = 不预警
- 备注、状态（启用/停用）

> 删除型号后，如果之后重新创建同名型号，系统会自动恢复之前的记录，历史单据引用不受影响。

**停用影响**：商品或型号停用后，销售订单、采购单、发货单的下拉中无法选择，且不参与库存预警计算。

## 销售员

维护销售员档案，与订单关联后用于提成计算和业绩统计。

**字段**：

| 字段 | 说明 |
| --- | --- |
| 姓名 | 必填 |
| 联系电话 | 选填 |
| 提成比例(%) | 必填，默认 40%，用于计算销售提成 |
| 关联账号 | 选填，绑定系统用户（必须具有销售角色），绑定后该用户可在「我的业绩」视图中看到自己的数据 |
| 备注 | 选填 |

**规则**：

- 一个系统用户只能绑定一个销售员
- 关联的用户必须具有「销售」角色，否则保存报错
- **停用影响**：停用后新订单无法选择该销售员；停用销售员的订单完成时**不计提提成**

## 快递公司

维护发货时可选的物流商（如 DHL、FedEx 等）。

**字段**：快递公司名称（必填，唯一）、备注。

**引用**：发货单创建时选择快递公司。停用后新发货单无法选择。

## 运输渠道

维护运输方式/渠道（如空运、海运、铁路、国际快递等）。

**字段**：渠道名称（必填，唯一）、备注。

**引用**：销售订单创建时选择运输渠道。停用后新订单无法选择。

## 成本类型

维护订单额外成本的分类。系统预置：物流成本、广告成本、营销成本、扣关成本、其他成本。

**字段**：成本名称（必填，唯一）、排序号、备注。

**引用**：销售订单的「成本信息」区块选择成本类型。**同一订单同一成本类型只能添加一次**，重复添加会报错。停用后新单据无法选择，历史成本记录不受影响。

## 常用联系人

维护客户侧常用联系人名称，便于录单时参考。

**字段**：联系人名称（必填，唯一）。

**操作**：支持编辑和删除（删除后不可恢复，但不影响历史订单）。列表按使用次数排序展示。

## 汇率管理

> 【图片占位：汇率管理列表页】

维护 USD→CNY 汇率数据，供全系统多币种金额折算使用。

### 列表与操作

- 搜索：源币种、生效日期范围
- 操作：新增汇率、编辑、删除、**同步汇率**（自动拉取最近 30 天的外部汇率数据）

### 新增汇率

**字段**：源币种（默认 USD）、目标币种（默认 CNY）、汇率（如 7.2000）、生效日期。

**唯一约束**：同一币种对 + 同一生效日期只能有一条记录，重复新增会提示已存在。

### 汇率取数规则

业务单据（订单、收款、采购、成本等）按**单据日期**自动查找汇率，查找顺序：

1. 精确匹配该日期的汇率记录
2. 未找到时系统自动从网络获取并保存
3. 仍未找到时取该日期之前最近的历史汇率
4. 再取该币种最新一条记录
5. 全部找不到 → 提示「未找到汇率数据，请先在汇率管理中维护」

> 如果创建单据时提示查不到汇率，请先在汇率管理页面手动新增对应日期的汇率，或点击「同步汇率」自动拉取。

### 默认汇率

系统参数中的「默认汇率（USD→CNY）」（默认 6.8）作为兜底值，仅在特殊场景使用。业务单据优先按上述链路取实际汇率。

## 提成管理

> 【图片占位：提成管理页（提成分录 Tab + 统计卡片）】

提成管理展示系统自动生成的提成分录与月度结算记录。**分录由系统自动生成，无需手工创建。**

### 提成计算规则

- 提成 = 销售利润（若为负则按 0）× 销售员提成比例
- 利润公式：订单金额 − 博主佣金 − 已退款 − 直接退款 − 产品成本 − 额外成本
- 提成比例取销售员档案设置（默认 40%），未设置时取系统参数中的「默认提成比例」
- 利润为负或为零时不产生提成

### 提成分录 Tab

**类型**：计提（绿色，订单完成时生成）/ 冲回（红色，退款/退货/订单重开时生成）。

**状态**：待结算（橙色）/ 已结算（蓝色，月度结算后标记）。

**顶部统计卡**（随筛选条件联动）：本月计提、本月冲回、本月净提成、待结算总额。

**触发时机**：

- 计提：订单变为「已完成」时（发货完成、收款完成、终止完成、退货后满足完成条件）
- 冲回：已完成订单发生退款/退货时差额重算；订单重新打开时全额撤销
- 补提：退款冲抵后利润反升时

### 月度结算 Tab

> 【图片占位：月度结算弹窗与结算列表】

**结算流程**：

1. 点击「月度结算」，选择要结算的月份
2. 系统自动汇总该月所有待结算分录，按销售员生成结算记录
3. 结算记录状态为「待确认」，核对后点击「确认发放」填写实际发放金额

**结算规则**：

- **不能结算当月**（当月尚未结束），只能结算历史月份
- **不能跳月**：必须按顺序结算（如 3 月未结算时不能结算 4 月）
- 同一月份不可重复结算
- **上月结余**：如果上月净提成为负数（冲回大于计提），负数结余滚入下月抵扣；正数不累计
- 净提成 = 本月计提 − 本月冲回 + 上月结余
- 月份归属按分录创建时间（即：跨月退款冲回记入退款当月，而非原订单所在月份）

**结算状态**：待确认 → 已发放（确认发放后）/ 已取消。
