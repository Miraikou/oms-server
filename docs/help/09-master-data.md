# 基础资料

基础资料是业务单据的支撑数据，建议在正式开单前完成维护。

## 供应商

- 维护供应商名称、联系人、电话、地址等
- 停用后不可被新采购单选择，历史单据不受影响

## 商品分类与商品

- **分类**：两级分类体系，用于商品归类
- **商品**：记录商品名称、编码、图片、描述等
- **型号**：商品下可挂多个型号（规格），每个型号可单独设置**最低库存预警阈值**（留空则使用全局阈值）
- 商品/型号停用后不参与销售、采购与库存预警

## 销售员

- 维护销售员信息，订单创建时关联
- 驾驶舱销售排行按销售员统计

## 物流相关

| 模块 | 用途 |
| --- | --- |
| 快递公司 | 发货时选择的物流商（DHL、FedEx 等） |
| 运输渠道 | 运输方式/渠道（空运、海运、快递专线等） |
| 费用类型 | 订单费用的分类（运费、关税、保险等） |

## 常用联系人

维护客户侧常用联系人，创建订单时可快速选择收货人信息。

## 汇率管理

- 维护 USD/CNY 汇率，供多币种金额折算使用
- 系统配置中可设置**默认汇率**，新建单据时自动带入，可在单据上覆盖

## 提成管理

- 按订单与规则计算销售员提成
- 系统配置中可设置默认提成参数
