# 发货管理

发货单记录每一次实际出库发货，与库存批次、成本核算直接关联。发货单号以 `FH` 开头。

> 【图片占位：发货管理列表页】

## 发货单列表

### 搜索条件

- 发货单号（模糊匹配）、订单号（精确匹配）、快递单号（模糊匹配）
- 快递公司下拉、发货日期范围

### 列表字段

| 列 | 说明 |
| --- | --- |
| 发货单号 | FH + 日期 + 流水号 |
| 订单号 | 关联的销售订单 |
| 快递公司 / 快递单号 | 物流信息 |
| 发货时间 | 发货日期 |
| 状态 | 固定为「已发货」（发货单一旦创建不可撤销） |
| 类型 | 正常发货 / 补发 |

> 发货单没有编辑、删除操作——发错货请通过「客户退货」流程处理，不要手工调整库存。

## 新建发货

> 【图片占位：新建发货弹窗（含 FIFO 批次预览区）】

### 入口

- 方式一：在订单列表对目标订单点击「发货」（自动带入订单，无需再选）
- 方式二：发货管理页点击「新建发货」，手动选择订单（仅列出进行中且未全部发货的订单）

### 发货类型

| 类型 | 说明 |
| --- | --- |
| 正常发货 | 常规发货，按明细销售单价计算收入 |
| 补发 | 对应「补发不退货」场景：需先在客户退货中创建「补发不退货」退货单，补发明细的销售金额按 0 计（纯成本单据） |

> 选择「补发」但订单没有补发不退货记录时，系统会拒绝并提示先创建退货单。

### 物流信息

快递公司（必选，在「快递公司管理」维护）、快递单号（必填）、发货日期（默认当天）、备注。

### 发货明细与 FIFO 批次预览

选择订单后，系统自动加载可发货明细（可发数量 > 0 的行），并默认填入全部可发数量：

- **可发数量** = 有效需求量（订购量 − 仅退款量）− 客户持有量（已发 − 已退）
- 每行的**预估成本**随「本次发货」数量输入实时联动重算
- 下方「FIFO 批次预览」展示实际分配的批次：`批次号: 数量×¥单位成本`，悬停可查看美元成本

**FIFO（先进先出）规则**：系统从最早入库的有效批次开始依次扣减，直到满足本次发货数量。库存不足时报错「库存不足：需要 X，可用 Y」，无法超发。

### 提交

- 至少填写一行发货数量才能提交
- 提交成功后库存实时扣减，生成库存流水，订单发货状态同步更新
- 如果订单因此变为「已完成」且关联了销售员，系统自动计提销售提成

## 发货详情

> 【图片占位：发货详情抽屉（FIFO 批次消耗明细展开状态）】

点击「详情」查看：

1. **基本信息**：发货单号、订单号、物流公司、单号、日期、类型
2. **汇总**：销售总额、产品总成本、毛利（正绿负红）
3. **发货明细表**：商品、型号、数量、批次号、销售单价/金额、产品成本、毛利
4. **FIFO 批次消耗明细**：折叠面板，按商品分组展示每个批次的消耗数量与成本单价

## 注意事项

- 发货会**实时扣减库存**并生成库存流水，操作前请确认数量
- 部分发货的订单可继续创建新的发货单，直至全部发完
- 同一商品存在多个库存批次时，系统优先扣减最早入库的批次
- 批次预览仅供核对，提交后后端按相同规则重新计算并落库，保证数据一致
- 发错货需退回的，请通过「客户退货」流程处理
