# 发货管理

发货单记录每一次实际出库发货，与库存批次、成本核算直接关联。

## 发货流程

发货必须基于销售订单创建：

1. 在订单列表对目标订单点击「发货」
2. 勾选本次发货的明细行并填写数量
3. 系统自动按 **FIFO（先进先出）** 从最早入库的批次开始分配库存
4. 确认预估成本后填写物流信息（快递公司、运单号）并提交

## FIFO 批次分配

- 同一商品存在多个库存批次时，系统优先扣减**最早入库**的批次
- 发货弹窗中的「批次预览」会展示每个批次被分配的数量与单位成本，且**随本次发货数量实时联动**
- 预估成本 = 各批次分配数量 × 批次单位成本之和（人民币）

> 批次预览仅供核对，提交后后端按相同规则重新计算并落库，保证数据一致。

## 发货单列表

- 支持按发货单号、关联订单号、发货状态、日期筛选
- 点击发货单可查看：发货明细、批次分配详情、物流信息、预估成本

## 注意事项

- 发货会**实时扣减库存**并生成库存流水，操作前请确认数量
- 部分发货的订单可继续创建新的发货单，直至全部发完
- 发错货需退回的，请通过「销售退货」流程处理，不要手工改库存
