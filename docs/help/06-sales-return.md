# 销售退货管理

已发货订单发生售后问题时，通过客户退货单处理。退货单号以 `TH` 开头，创建后不可修改。

> 【图片占位：客户退货管理列表页】

## 退货类型

系统支持四种退货类型，**恢复库存与退回货款由类型自动决定**，不可手工更改：

| 类型 | 场景 | 恢复库存 | 退回货款 |
| --- | --- | --- | --- |
| 退货退款 | 客户退回货物并退款 | 是 | 是 |
| 退货换货 | 客户退回货物，后续补发新货 | 是 | 否 |
| 仅退款 | 货物不退回，仅退还货款 | 否 | 是 |
| 补发不退货 | 货物损坏等，直接补发新货 | 否 | 否 |

> 「退货换货」和「补发不退货」是为后续补发做准备的类型：退货换货会把退回的货恢复到可用库存；补发不退货则从现有可用库存中按 FIFO（先进先出，即先入库的货先发出）扣减补发。补发时在发货管理选择「补发」类型即可。

## 退货单列表

### 搜索条件

- 退货单号（模糊匹配）、订单号（精确匹配）、退货日期范围

### 列表字段

退货单号、订单号、退货日期、退货类型、恢复库存（是/否）、退回货款（是/否）、退回货款金额、退货额外成本、退货原因。

## 新建退货单

> 【图片占位：新建客户退货弹窗（退货类型选择 + 可退货商品表）】

### 入口

- 方式一：在订单列表对已发货订单点击「退货」（自动带入订单）
- 方式二：客户退货管理页点击「新建客户退货」，手动选择订单（仅列出非已取消且已发过货的订单）

### 操作步骤

1. **选择退货类型**（默认「仅退款」）——选择后下方两个开关自动联动，无需手工设置
2. 填写退货日期（默认当天）
3. 如有退货产生的额外成本（如退货运费），填写金额与币种
4. 在「可退货商品」表中填写每行的退货数量（不超过可退数量）
5. 填写退货原因、备注后提交

### 可退数量

可退数量 = 发货明细数量 − 该明细历史退货累计。已退完的明细不会出现在列表中。

### 退回货款金额

退货退款/仅退款类型下，退回货款金额**自动计算**（只读）：

> 退回金额 = 各行（退货数量 × 对应销售单价）的合计

按订单币种计算，同时展示双币种。退款方式可选填。

## 提交后的系统处理

### 库存恢复（退货退款 / 退货换货）

- 按**原发货批次**依 FIFO 顺序恢复可用库存（已耗尽的批次会重新变为有效）
- 退回的货统一回到可用库存，后续补发/换货发货与其他订单一样按 FIFO 扣减
- 同步扣减对应发货明细的产品成本，毛利随之重算
- 生成库存流水（业务类型：客户退货入库）

### 退款（退货退款 / 仅退款）

- 系统自动生成退款单（`TK` 前缀），按**订单原始汇率**换算
- 累计退款金额不能超过已收金额
- 退货退款**不扣减订单已收金额**（保持收款记录完整），记入独立的「退货退款/仅退款」字段
- 退款金额参与利润计算，并触发提成差额重算

### 退货成本

填写了退货额外成本时，系统自动归入订单的「客户退货成本」类型（同一订单累加），参与利润计算。

### 订单状态与数量影响

| 类型 | 数量影响 | 可能的状态变化 |
| --- | --- | --- |
| 退货退款 | 降低客户持有量 + 降低有效需求量 | 可能使进行中订单直接满足完成条件 |
| 退货换货 | 降低客户持有量 | 可能使已完成订单重新打开（等待补发） |
| 仅退款 | 不影响数量 | 不影响发货/完成状态 |
| 补发不退货 | 降低客户持有量 | 可能使已完成订单重新打开（等待补发） |

## 退货详情

点击「详情」查看：退货单号、订单号、日期、类型、恢复库存、退回货款及金额、退货额外成本、原因、备注，以及退货明细表（发货单号、商品、型号、退货数量、销售单价、是否恢复库存）。

## 注意事项

- 只有**已发货**（部分或全部）的非取消订单可以发起退货；未发货订单请直接取消
- 退货数量不能超过可退数量（发货数量 − 已退数量）
- 同一发货明细行不可在一张退货单中重复出现，请合并数量
- 退货换货/补发不退货类型不支持退款，如需退款请选择退货退款或仅退款
