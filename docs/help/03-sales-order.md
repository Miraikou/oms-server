# 销售订单管理

销售订单是系统的核心单据，承载从成交到完成的全过程。订单号以 `SO` 开头，由系统自动生成。

> 【图片占位：订单列表页全貌（含搜索区与状态标签）】

## 订单状态说明

订单有三个并行的状态维度：

| 维度 | 取值 | 含义 |
| --- | --- | --- |
| 订单状态 | 进行中 / 已完成 / 已取消 | 订单生命周期 |
| 发货状态 | 待发货 / 部分发货 / 全部发货 | 履约进度 |
| 收款状态 | 未收款 / 部分收款 / 已收款 | 回款进度 |

**自动流转规则**：

- 全部发货且全部收款后，订单自动变为「已完成」
- 已完成的订单如果发生退货换货导致发货状态下降，会重新打开为「进行中」
- **已取消的订单状态永远不会再变更**

> 下单时不占用库存（无预留模型），库存在实际发货时才扣减。

## 列表与筛选

### 我的订单 / 全部订单

拥有全局查看权限时，列表上方可切换「我的订单」与「全部订单」：

- **我的订单**：只显示当前账号关联的销售员的订单
- **全部订单**：显示所有订单，并额外出现「销售员」筛选下拉

无权限时固定为「我的订单」视图。

### 搜索条件

- 订单号（模糊匹配）、销售员（仅全部订单视图）
- 订单状态（单选）、发货状态（**多选**）、收款状态（**多选**）
- 下单日期范围

### 列表字段说明

| 列 | 说明 |
| --- | --- |
| 订单总金额 | 未减去退款金额 |
| 退货退款/仅退款 | 红色显示，来自客户退货的退款累计 |
| 博主佣金 | = 订单金额 × 佣金比例 |
| 销售利润 | = 订单金额 − 博主佣金 − 已退款 − 直接退款 − 产品成本 − 额外成本；正数绿色、负数红色 |
| 销售提成 | = 销售利润 × 提成率（利润为负时按 0 计） |

> 产品成本仅统计**已发货**商品的 FIFO 采购成本（收付实现制），未发货部分不计入。

### 操作按钮

| 按钮 | 出现条件 |
| --- | --- |
| 详情 | 始终可见 |
| 编辑 | 进行中 且 待发货 |
| 收款 | 进行中 且 未收齐 |
| 退款 | 进行中 且 有已收款 |
| 发货 | 进行中 且 未全部发货 |
| 退货 | 非已取消 且 已发过货 |
| 取消 | 进行中 |
| 终止 | 进行中 且 部分发货 |

## 创建订单

> 【图片占位：新建订单弹窗（基本信息 + 商品明细 + 收款 + 成本四个区块）】

点击「新建订单」，弹窗分为四个区块：

### 基本信息

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 销售员 | 是 | 仅显示启用的销售员 |
| 客户名称 | 是 | 最多 100 字 |
| 下单日期 | 是 | 默认当天，影响汇率取值 |
| 运输渠道 | 是 | 如空运、海运等，在「运输渠道管理」维护 |
| 交易方式 | 是 | 在「字典管理」中维护（TRADE_TYPE） |
| 博主佣金比例(%) | 是 | 默认 5%，用于计算博主佣金 |
| 币种 | 是 | 人民币(CNY) / 美元(USD)，默认 USD |
| 汇率 | — | **自动查询，不可手工修改**：按下单日期取 USD→CNY 汇率 |
| 备注 | 否 | — |

> 切换下单日期或币种时，汇率会自动重新查询。如果提示查不到汇率，请先在「汇率管理」中维护对应日期的汇率。

### 商品明细

至少一行商品。每行：商品（必选）、型号（可选，随商品联动）、数量（正整数）、单价。金额 = 数量 × 单价，实时计算只读。

### 收款信息（可选）

新建时可直接登记首笔收款：选择「已收款」后填写收款金额、日期、方式等。

> 收款金额不能超过货物总金额（各明细数量×单价之和），否则无法提交。

### 成本信息（可选）

可登记订单相关的额外成本（运费、广告费等）：

- 成本类型从「成本类型管理」选取，**同一订单同一成本类型只能添加一次**
- 选择非人民币币种时自动查询当天汇率换算

## 编辑订单

**仅「待发货」状态的订单可以编辑。** 部分发货或全部发货后，订单不可再修改。

编辑时的限制：

- 已发货后：销售员、运输渠道、币种不可改，商品明细不可增删
- 已收款后：博主佣金比例不可改
- 修改商品明细后，如果已收金额超过新的货物总金额，系统会拒绝保存并提示先退款

## 发货

在操作列点击「发货」进入发货流程，详见「发货管理」章节。要点：

- 系统按 FIFO（先进先出）自动分配库存批次，库存不足无法超发
- 发货弹窗中可实时预览本次发货的预估成本与批次分配
- 部分发货后订单变为「部分发货」，可多次发货直至全部发完

## 收款与退款

在操作列点击「收款」或「退款」，详见「收款管理」章节。要点：

- 收款币种固定为订单币种，汇率按收款日期自动查询
- 退款会扣减已收金额，可能使收款状态回退
- 后续收款会优先冲抵此前的直接退款，冲抵后利润相应恢复

## 退货

在操作列点击「退货」进入客户退货流程，详见「销售退货管理」章节。支持四种类型：退货退款、退货换货、仅退款、补发不退货。

## 取消订单

取消有严格限制，避免账实混乱：

- **已发货（部分或全部）的订单不可取消**，请走退货流程
- **已有收款的订单不可取消**，请先完成退款
- 已完成、已取消的订单不可再操作

> 取消后如果有需要退款的金额，系统会提示前往收款管理处理。

## 终止订单

终止用于「部分发货后客户弃购剩余未发部分」的场景：

- **仅部分发货的进行中订单可以终止**（待发货请用取消，全部发货无需终止）
- 终止时系统自动计算弃购明细，将订单明细数量收缩为已发货数量，订单总金额同步减少
- 如果已收款超过终止后的订单金额，系统**自动生成退款单**退还超出部分
- 终止后如果满足「全部发货 + 已收款」条件，订单会流转为已完成

> 此操作不可撤销，弹窗中会展示弃购明细和预计退款金额，请确认后再提交。

## 订单详情

> 【图片占位：订单详情抽屉（利润概览区块）】

点击订单号或「详情」打开详情抽屉，包含以下区块：

1. **基本信息**：订单号、客户、日期、交易方式、金额、状态等
2. **商品明细**：含每行的已发货、已退货、仅退款、**可发货**数量
3. **收款记录 / 退款记录 / 退货记录**：有记录时显示
4. **利润概览**：
   - 产品成本 = 已发货商品的 FIFO 采购成本合计
   - 额外成本 = 订单成本明细合计
   - 订单净额 = 订单金额 − 博主佣金 − 退货退款/仅退款 − 直接退款
   - 销售利润 = 订单金额 − 博主佣金 − 已退款 − 直接退款 − 产品成本 − 额外成本
   - 利润率 = 销售利润 ÷ 订单金额 × 100%
5. **成本明细**：进行中订单可在此添加/编辑/删除成本（已完成或已取消则不可操作）

> 直接退款净额 = 直接退款累计 − 后续收款冲抵。后续收款会优先冲抵此前的直接退款，冲抵后利润相应恢复。

## 提成

订单变为「已完成」时，系统自动为关联销售员计提提成：

- 提成 = max(0, 销售利润) × 销售员提成比例
- 利润为负或为零时不计提
- 销售员已停用时不计提
- 订单因退货/退款重新打开时，已计提的提成会被全额撤销；已完成状态下发生退款会差额重算

提成分录与结算详见「基础资料 → 提成管理」。
