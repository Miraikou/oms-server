# 销售订单管理

销售订单是系统的核心单据，承载从成交到完成的全过程。

## 订单状态说明

订单有三个并行的状态维度：

| 维度 | 取值 | 含义 |
| --- | --- | --- |
| 订单状态 | 进行中 / 已完成 / 已取消 | 订单生命周期 |
| 发货状态 | 待发货 / 部分发货 / 全部发货 | 履约进度 |
| 收款状态 | 未收款 / 部分收款 / 已收款 | 回款进度 |

**自动流转规则**：全部发货且全部收款后，订单自动变为「已完成」。

## 创建订单

1. 订单列表点击「新建订单」
2. 填写基本信息：客户、销售员、交易方式、币种、收货地址等
3. 添加商品明细：选择商品与型号、数量、单价
4. 可登记预估费用（运费等，关联费用类型与运输渠道）
5. 保存后订单进入「进行中 / 待发货 / 未收款」

> 订单金额支持美元（USD）与人民币（CNY）双币种，按系统配置的默认汇率折算，可在单据上调整。

## 发货

1. 在订单操作列点击「发货」
2. 选择本次发货的订单明细与数量
3. 系统按 **FIFO（先进先出）** 自动分配库存批次，并展示本次发货的预估成本（随发货数量实时联动）
4. 填写快递公司与运单号后提交

部分发货后订单变为「部分发货」，可多次发货直至全部发完。

> 库存不足的批次无法超发；发货会实时扣减库存并生成库存流水。

## 收款

1. 在订单操作列点击「收款」
2. 选择收款币种（USD/CNY）、填写金额与收款方式
3. 提交后累计已收金额，收款状态随之更新

## 退货与退款

- 已发货订单发生退货，请使用「销售退货」模块创建退货单（区分仅退款/退货退款）
- 退货入库后系统自动恢复对应库存

## 取消订单

取消有严格限制，避免账实混乱：

- **已发货（部分或全部）的订单不可取消**，请走退货流程
- **已有收款的订单不可取消**，请先完成退款
- 取消后订单状态变为「已取消」，不再参与待办统计

## 查看与筛选

- 列表支持按订单号、状态（多选）、销售员、日期范围等筛选
- 点击订单号查看完整详情（明细、发货记录、收款记录、费用、操作轨迹）
