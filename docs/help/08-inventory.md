# 库存管理

库存管理提供实时库存视图、低库存预警、批次追溯、库存调整与全链路流水审计。涉及三个页面：当前库存、库存流水、库存调整记录。

> 【图片占位：当前库存页（树形表格展开型号行）】

## 库存结构说明

系统从三个层面管理库存：

1. **库存汇总**：按「商品 + 型号」汇总的当前库存数量，页面直接展示
2. **库存批次**：每次入库生成一个批次，记录数量与成本，是发货成本核算的依据
3. **库存流水**：每一次库存变动的完整记录，只读、不可修改删除

> 下单时不会占用库存，只有实际发货时才会扣减。因此页面上看到的「可用库存」就是实际可以发出的数量。

## 当前库存

### 树形列表

库存按「商品 → 型号」两级树形展示：

- **父行（商品）**：显示型号数量，如有低库存型号会追加红色提示
- **子行（型号）**：显示可用库存（低于阈值时红色，正常时蓝色）、实际库存、更新时间
- 子行操作列点击「批次」可查看该商品型号的批次明细

### 搜索与筛选

- 商品下拉（精确筛选）
- **「仅低库存」开关**：只显示触发预警的商品型号（驾驶舱库存预警卡片跳转后自动开启）

分页按商品维度，每页 10 个商品，展开查看型号。

### 库存预警阈值

判定规则：**可用库存 ≤ 阈值** 时触发预警。

阈值取值优先级：

1. **型号级阈值**（商品型号上设置的最低库存预警值）优先
2. 型号未设置（留空）时，使用**全局阈值**（系统参数「库存预警阈值」，默认 0）

型号阈值的特殊含义：

- 留空 = 使用全局阈值
- 填 0 = 库存为 0 时才预警
- 填负数 = 不预警

> 停用商品、停用/已删除的型号不参与预警计算。

> 系统还支持库存预警邮件通知：管理员在「系统参数」中开启开关并配置收件邮箱后，库存首次从高于阈值降至阈值以下时会自动发送邮件提醒（库存恢复后再次降至阈值以下会再次通知）。详见「系统管理 → 系统参数」。

### 一致性校验

页面右上角「一致性校验」按钮：检查系统记录的库存总数与各批次明细之和是否吻合。

- 全部匹配 → 提示通过
- 有不一致 → 页面下方展示不一致记录明细，请及时排查（通常由异常操作导致，可联系管理员执行库存校准）

## 批次详情

> 【图片占位：批次详情抽屉（按入库单分组）】

点击型号行的「批次」打开批次抽屉，按入库单分组展示：

- 父行：入库单号、入库日期、批次数、可用合计
- 子行：批次号（可点击查看该批次流水）、型号、可用数量、库存数量、状态（有效/耗尽）

没有来源入库单的批次（库存调整或客户退货恢复产生的）归入「其他入库（调整/退货）」分组。

点击批次号可进一步查看**该批次的全部流水**：业务类型、变化类型、数量、单价、总成本、前后可用量。

## 库存流水

> 【图片占位：库存流水列表页】

「库存流水」页面是只读审计页，记录库存的每一次变化，**不可修改、不可删除**。

### 搜索条件

- 商品下拉、批次号（精确匹配）、业务类型

### 业务类型

| 类型 | 说明 |
| --- | --- |
| 采购入库 | 采购到货入库增加 |
| 销售发货 | 发货扣减 |
| 客户退货入库 | 退货恢复库存 |
| 采购退货出库 | 采购退货扣减 |
| 库存调整 | 盘点调整 |

### 变化类型

入库（绿）/ 出库（红）/ 调整（紫，悬停可查看调整单号）。

列表展示：商品、型号、批次号、业务类型、变化类型、数量（正绿负红）、单价、总成本、前可用、后可用、创建时间。

## 库存调整

盘点发现账实不符时，通过库存调整单修正。调整单号以 `KC` 开头。**提交即生效，无审批流程**，请谨慎操作。

> 【图片占位：新建库存调整弹窗】

### 新建调整

1. 选择调整原因：

| 原因 | 说明 |
| --- | --- |
| 盘盈 | 盘点时实际多于账面，增加库存 |
| 盘亏 | 实际少于账面，扣减库存 |
| 系统修正 | 非盘点的人工修正（如入库数量录错），可增可减 |
| 其他 | 预留扩展 |

2. 添加调整明细（可多行）：
   - 商品（必选）、型号（可选）
   - **批次（可选）**：留空 = 自动处理；指定批次则只调整该批次
   - **调整数量**：正数 = 增加，负数 = 减少
   - **成本来源**（增加且未指定批次时必填）：
     - 近一年加权平均成本：按近一年入库批次加权计算
     - 剩余库存加权平均成本：按当前有库存的批次加权
     - 最新采购记录成本：取最近一次入库单价
     - 手动输入：自行填写单价与币种
   - 选择成本来源后可点击「估算」预览单价

3. 提交后库存即时更新，生成库存流水（业务类型：库存调整）

### 调整规则

| 场景 | 处理方式 |
| --- | --- |
| 增加 + 指定批次 | 该批次数量增加（耗尽批次可复活），成本沿用批次原成本 |
| 增加 + 未指定批次 | 生成新批次（来源：库存调整），成本按所选成本来源计算 |
| 减少 + 指定批次 | 仅从该批次扣减，不足时报错 |
| 减少 + 未指定批次 | 按 FIFO（先进先出）从最早批次依次扣减 |

### 调整记录列表

搜索：调整单号（精确匹配）、业务类型。详情展示调整明细（商品、型号、批次、调整数量、单价）。

## 常见场景指引

- **库存不足无法发货**：说明可用库存低于发货需求量，请先采购入库或检查是否有未处理的入库
- **账面与实物不符**：先通过批次详情和流水追溯变动来源，确认后用库存调整修正
- **预警阈值不合理**：在商品型号管理中调整型号级阈值，或在系统参数中调整全局阈值
