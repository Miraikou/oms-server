# 系统管理

系统管理模块面向管理员，负责用户、权限、字典与运行配置。包含：用户管理、角色管理、菜单管理、字典管理、操作日志、登录日志、系统参数，以及个人信息页。

## 用户管理

> 【图片占位：用户管理列表页】

### 列表与搜索

搜索：账号/姓名（模糊匹配）、状态（启用/停用）。

列表字段：账号、姓名、手机号、邮箱、角色（标签展示）、状态、最后登录时间、创建时间。

### 新增用户

| 字段 | 说明 |
| --- | --- |
| 账号 | 必填，创建后不可修改 |
| 密码 | 必填，至少 6 位 |
| 姓名 | 必填 |
| 手机号 | 选填，11 位 |
| 邮箱 | 选填，需符合邮箱格式 |
| 角色 | 选填，可多选（决定该用户能看到哪些菜单和操作） |
| 状态 | 默认启用 |
| 备注 | 选填 |

> 账号不可重复。未分配角色的用户登录后将看不到任何菜单。

### 编辑用户

可修改姓名、手机号、邮箱、角色、状态、备注。**账号不可修改。**

> 编辑时如果清空角色选择，将移除该用户的全部角色（有意的安全管理操作）。

### 重置密码

点击操作列「重置密码」，输入新密码（至少 6 位）即可。不校验旧密码，重置后用户下次登录使用新密码。

### 停用影响

- 停用后该用户无法登录（提示「用户不存在或已停用」）
- 已登录的会话在凭证到期后无法续期，等效强制下线
- 停用没有额外限制（不会阻止停用管理员账号，请谨慎操作）

## 角色管理

> 【图片占位：角色分配权限弹窗（菜单树勾选）】

### 基本概念

角色是权限的集合：给用户分配角色 → 角色关联菜单和按钮 → 用户获得对应的可见菜单与可操作按钮。

系统预置 6 个角色：超级管理员（SUPER_ADMIN）、老板（BOSS）、销售（SALES）、采购（PURCHASER）、仓库（WAREHOUSE）、财务（FINANCE）。

### 新增/编辑角色

**字段**：角色名称（必填，唯一）、角色编码（必填，唯一，创建后不可修改）、状态、备注。

### 分配权限

点击「分配权限」打开菜单树弹窗，勾选该角色可访问的菜单和按钮：

- 树节点包含目录、菜单、按钮三级
- 按钮节点显示权限码（如 `order:create`）
- 保存后**全量覆盖**该角色的权限配置

### 删除角色

已被用户引用的角色无法删除（提示「该角色已被用户引用，无法删除」）。需先解除所有用户的该角色关联。

### 超级管理员特权

SUPER_ADMIN 角色拥有特殊待遇：

- 所有接口权限校验直接放行，不逐一比对权限码
- 自动获得全部启用菜单和按钮权限
- 驾驶舱默认全局视图

## 菜单管理

> 【图片占位：菜单管理树形表格】

维护左侧导航的菜单树，以树形表格展示（无分页，默认全部展开）。

### 菜单类型

| 类型 | 说明 | 特有字段 |
| --- | --- | --- |
| 目录 | 一级分组（如「业务操作」「仓储采购」） | 路由路径、图标 |
| 菜单 | 具体页面（如「订单管理」） | 路由路径、图标 |
| 按钮 | 页面内的操作权限点 | **权限标识**（如 `order:create`） |

### 新增/编辑菜单

- 上级菜单：树形选择（按钮类型不能作为父节点）
- 菜单类型、菜单名称（必填）
- 按钮类型：只需填写权限标识
- 目录/菜单类型：填写路由路径（如 `/orders`）和图标
- 排序号、是否显示、状态

### 删除限制

- 有子菜单时无法删除（需先删除子节点）
- 被角色引用时无法删除（需先在角色中取消勾选）

### 权限码说明

按钮型菜单的「权限标识」是全系统权限控制的唯一来源。前端按钮根据当前用户拥有的权限码决定是否显示。如果用户看不到某个按钮，请检查其角色是否勾选了对应的按钮权限。

## 字典管理

维护系统动态枚举值，采用「字典类型 → 字典项」两级结构。

### 字典类型

搜索：编码/名称、状态。列表字段：字典编码、字典名称、状态、创建时间、备注。

**字段**：字典编码（必填，如 `PAYMENT_METHOD`，创建后不可修改，唯一）、字典名称（必填）、备注。

### 字典项

点击类型的「查看」进入字典项管理抽屉：

**字段**：字典值（必填，如 `ALIPAY`）、字典标签（必填，显示文本如「支付宝」）、排序号、状态、备注。

**唯一约束**：同一字典类型下字典值不可重复。

### 注意事项

- 字典类型和字典项**只能停用/启用，不能删除**
- 停用后业务页面的对应下拉选项自动隐藏
- 系统预置了 PAYMENT_METHOD（支付方式）和 TRADE_TYPE（交易方式）两个字典类型，字典项需自行维护
- 订单表单中的「交易方式」「收款方式」下拉即来自这两个字典

## 操作日志

只读查询页，记录系统中的关键写操作。

### 记录范围

- 记录所有 POST/PUT/PATCH/DELETE 请求（新增、修改、删除操作）
- GET 查询不记录；登录/刷新 Token 不记录（登录走登录日志）
- 请求体中的密码、文件等敏感字段自动脱敏

### 搜索与查看

筛选：模块、操作类型、操作人 ID、时间范围。

> 模块和操作类型筛选为精确匹配，如果按条件查不到记录，建议直接按时间范围或操作人查询。

点击「详情」查看：操作内容（请求参数 JSON）、请求地址、IP、操作结果（成功/失败）。

## 登录日志

只读查询页，记录每次登录尝试（成功与失败）。

筛选：登录账号（模糊匹配）、登录结果（成功/失败）、时间范围。

列表字段：登录账号、登录 IP、浏览器、登录结果、登录时间。

> 登录地点字段当前未启用。登出和自动续期不记录登录日志。

## 系统参数

集中管理全局配置参数，修改后**立即生效**（无需重启）。

### 预置参数

| 参数键 | 名称 | 默认值 | 作用 |
| --- | --- | --- | --- |
| DEFAULT_COMMISSION_RATE | 默认提成比例（%） | 40 | 销售员未设提成比例时的兜底值 |
| LOW_STOCK_WARNING | 库存预警阈值 | 0 | 型号未单独设置阈值时的全局预警线 |
| DEFAULT_EXCHANGE_RATE | 默认汇率（USD→CNY） | 6.8 | 汇率查询全部未命中时的兜底值 |

### 操作

- 搜索：参数键/名称
- 可新建参数、编辑参数值（参数键创建后不可修改）
- **无删除功能**（预置参数不可删除）

> 值类型标注（STRING/NUMBER/BOOLEAN/JSON）仅供识别参考，系统按文本存储。

## 个人信息

点击右上角头像 →「个人信息」进入，包含两个 Tab：

### 基本信息

- 查看：用户名（只读）、角色标签
- 可修改：**手机号、邮箱**（姓名和账号不可在此修改）
- 手机号需符合国内 11 位格式，邮箱需符合标准格式

### 修改密码

输入旧密码、新密码（至少 6 位）、确认新密码。

> 修改成功后系统会**强制登出**，需使用新密码重新登录。
